### Base de données (Vercel Postgres / Neon-backed)
**16 tables migrées** :
users, accounts, sessions, verificationTokens, orgs, org_members, agent_instances, workflows, workflow_runs, tasks, audit_logs, memory_entries, integrations, approvals, projects, messages

### Sécurité
- AES-256-GCM (Web Crypto) pour les tokens OAuth au repos
- HMAC-SHA256 pour les state nonces OAuth (TTL 10 min)
- SHA-256 chaîné pour audit logs (tamper-proof)
- JWT sessions (Auth.js v5, 30j max)
- bcryptjs (12 rounds) pour les passwords

## Le combat technique de cette session

**Problème** : sur Vercel Hobby, les routes nodejs runtime hangaient à HTTP 000 (timeout 30s+) sans aucun logging d'erreur côté Vercel. Reproduction systématique sur les routes qui import `@/auth`, mais la même fonction marchait à un autre path (/api/diag/agents-clone OK, /api/agents KO).

**Tentatives qui n'ont rien réglé** :
- Middleware edge-safe ✗
- DrizzleAdapter retiré ✗
- Switch neon-serverless Pool → neon-http ✗
- maxDuration=60 global ✗
- Lazy-load des heavy modules ✗
- Force-new redeploy + project rename ✗
- Path renames (/api/v1/*) ✗ (premiers deploys hangs aussi)

**Solution finale** : conversion de toutes les API en **runtime "edge"** + remplacement de `node:crypto` par **Web Crypto API** dans audit, oauth, lib/crypto, lib/agents/tools, lib/memory. Edge runtime a un cold-start très rapide (<200ms) et n'a pas le bug nodejs Hobby.

## Pour ton usage demain

### 1. Créer ton compte super-admin
```
1. Va sur https://axiyon-nine.vercel.app/signup
2. Email = celui qui correspond à SUPER_ADMIN_EMAIL dans Vercel env
3. Le compte sera marqué isSuperuser=true automatiquement
```

### 2. Connecter Gmail (premier provider)
1. Suis `docs/SETUP_OAUTH.md` section "Google Workspace"
2. Crée l'app OAuth chez Google → récupère `GOOGLE_CLIENT_ID` et `GOOGLE_CLIENT_SECRET`
3. Ajoute-les dans Vercel → Project axion-os → Settings → Environment Variables → Production
4. Redéploie (auto via push, ou manuellement)
5. Va sur `/dashboard/integrations` → clique "Connecter" sur Google
6. Tu seras redirigé vers Google pour autoriser → retour avec carte verte

## Email transactionnel (Resend)

**Configuré** : `RESEND_API_KEY` (clé fournie). Sender courant : `Axion <onboarding@resend.dev>`.

**Limite Resend free tier** : avec `onboarding@resend.dev`, les emails ne sont livrés qu'aux adresses **vérifiées sur ton compte Resend**. Pour livrer à n'importe quelle adresse en prod :

1. Va sur https://resend.com/domains
2. Ajoute un domaine que tu possèdes (ex : `axion.app`, ou `pirabellabs.com`)
3. Configure les DNS demandés (TXT/MX/CNAME)
4. Une fois le domaine "Verified", change `RESEND_FROM` sur Vercel vers `Axion <noreply@ton-domaine.com>`

**Fallback intégré** : si l'envoi Resend échoue, l'API `/api/v1/auth/password-reset/request` retourne tout de même `resetUrl` dans la réponse. La page `/reset-password` l'affiche directement avec un lien cliquable — l'utilisateur n'est jamais bloqué.
